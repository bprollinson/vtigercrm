3. Bug Fixes

We have fixed about 1150+ issues in the current release. Following list will give you a better idea on the issues addressed in various modules.

Top 10 - Bug Fixes:

  • 3055: Data integrity between various modules related to Accounts deletion. The following scenarios are handled in 5.0.3:
    • Delete account related potentials
    • Delete account related quotes
    • Delete the relation between Account-Contacts
    • Delete the relation between Account-Tickets
    • Delete the relation between Account-Activities
    • Delete account related notes & attachments
    • Not to delete account related Invoice and Sales Orders
  • 2933, 3842: During lead conversion, PO box and Mobile Number fields are not transferred to account & contact.
  • Data level security for Reports & Dashboards
  • 3353: In Edit View of the quote, Product discount & Tax values are corrected to last 2 digits after the decimal point. However, in Detail View more than 2 digits are displayed after decimal point. For example: In edit view the values are like 7834.90 In detail view the values are like 7834.89672
  • 3867: Interchange of Shipping & Billing addresses.
  • 3944: Under Trouble Tickets, if you create a Note (but do not fill in the "related to" ex: leads, accounts, sales order etc). You then save the note. It is attached to the Trouble Ticket. However, if you edit the Note again and fill in the "related to" field, the Note disappears from the list of Notes attached to the Trouble Ticket. If the "related to" was filled in on the first place, Note remains attached to the Trouble Ticket, which is an expected behavior.
  • 3533: Page layout related issues in FAQ detail view. Edit, Duplicate, and Delete buttons are not present at bottom in Detail view of FAQ module.
  • 3618 - Issue related to field-level security for activity type.
  • 2533 - Issue related to time format of the users not set properly. For example, one user follows AM/PM time where as other follows military time, then system always reset to AM/PM time.
  • The issues related to Quick edit of fields in CRM modules.
  • 3016: Internationalization support across the product.

For a complete list of bug fixes, please refer: vtiger Bug Tracker

4. Known Issues

We have fixed many more issues in the present release. However at the end-of-the-day we have some known issues left open, which will be addressed in the subsequent releases.

Top 10 - Know issues:

  • 3235: If mail contains more than one 'To' email ID. In this case, only first email ID in the 'To' list will be displayed. The other email IDs will be truncated.
  • 3832: Issue in displaying pick list values while using UTF8 character encoding.
  • 3365: It is not possible to use import wizard for products module that contains file attachments.
  • 3791: Issue in filtering reports data while using description type fields.
  • 3885: Base currency name will not display in Detail View for the currency type fields.
  • 2576: After customizing pick list values with ASCII characters, some of the pick list values are not displayed in quick create forms.
  • 2051: There are issues while using accents and quotes in pick list values
  • 3919: In Leads and Contacts module, salutation field will not display, if you are hiding the First name field.
  • 3728: The incoming message attachments (images and files) cannot be forwarded to another email ID.
  • 3574: Internationalization support is not available for vtiger Outlook Plugin.

For a complete list of known issues, see: vtiger Bug Tracker

5. Limitations

  • In future release, week view UI will be changed, so all the events are displayed at start time of the event.
  • If events start time is lesser than the Calendar start time, it will not be displayed in the calendar view.
  • Currency symbol will not be displayed for amount type fields except for potential amount.
  • Field-level access control is not implemented for Import and Export utilities
  • After mail merge template is uploaded to vtiger CRM, even if you change the field level security, it will be reflected in template
  • You cannot rename the reports.
  • Users at high-level hierarchy cannot view the data shared among their subordinates.
  • Users at high-level hierarchy cannot view the data shared among their subordinates' groups.
  • Users in various sub groups can access data of their parent groups but cannot access the data of their own sub groups.

    For example, A is a parent group, B is a sub group of A , and C is a sub group of B , then B can access data pertaining to A group, but cannot access the C sub-group data.
  • After converting lead to potential, campaign is not associated to the potential and the campaign source field will be blank. Due to this implementation, it is not possible to calculate success (ROI) of each campaign.
  • While adding a new potential in the CRM system, account name is a mandatory field (B2B). During this process, if you are planning to associate potential with some of the contacts related to potential-account, search the contacts Look-up page by the required account name before selecting contacts. Otherwise, there is a chance of associating contacts related to other accounts.
  • Web mail has been tested in some of the common scenarios.
  • While synchronizing contacts from vtiger CRM to Outlook, if the account name starts with lowercase, it adds to Outlook in Uppercase. For example vtiger from vtiger CRM will be converted to Vtiger . Hence there will be update of contacts every time.

6.Upgrading Procedure

This section is only for the existing vtiger CRM users. If you are planning to use vtiger CRM first time, please skip this section.

Important Note:

  • We recommend users to take a backup of the existing database and source code before migrating to vtiger CRM 5.0.3.
  • If you have customized the source code and database, please contact your developers or write to us at info@vtiger.com so that we can suggest a proper migration path.

Step 1: Copy vtiger CRM 5.0.3 - Patch files in to your vtiger CRM root directory.

Step 2: Migrate your database from earlier version to the latest version.

To upgrade from v4.2.3, 5.0.0, 5.0.1, 5.0.2 to the v5.0.3

  1. Select Settings --> Other Settings --> Migration Wizard
  2. In the Migration Wizard, select the options according to your requirements
  3. It will take some time to migrate the data. Kindly do not kill the browser during migration process.

Step 4: Remove some of the unwanted files and directories.

For a better security of the source files, we have reorganized some of the files location. Hence it is better to remove the files listed below permanently. Please take necessary precautions, while removing files.
